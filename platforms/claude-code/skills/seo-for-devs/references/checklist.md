# SEO Checklist for Developer Articles

## Must-Have

- [ ] Title: 50-60 chars, includes primary technology/problem
- [ ] Meta description: 120-160 chars, unique per article
- [ ] Canonical URL: set in frontmatter
- [ ] H1: one per page, matches or paraphrases title
- [ ] At least one code block (signals technical content)

## Good to Have

- [ ] Internal links: at least one link to another post on same domain
- [ ] Image alt text (if images are used)
- [ ] Date in frontmatter (search engines use freshness signals)

## Dev.to / Hashnode Specific

- [ ] Tags: 3-4 relevant tags (dev.to max is 4)
- [ ] Cover image: optional but boosts click-through on feeds

## Skip (over-engineering for a dev blog)

- Keyword density targets
- Schema markup (beyond what platforms add automatically)
- Backlink outreach
- Multiple H2/H3 nesting strategies
